## Phase 2 — Mobile (Expo Go + Supabase)

> **Why Expo Go over PWA on iPhone:** Chrome on iOS is WebKit under the hood — same limitations as Safari. JS timers die when the tab is backgrounded, so PWA notifications never fire on a locked phone. Expo Go has its own APNs certificate, so real push notifications work for free, no Apple Developer account required (personal use only).

## Architecture Reference

```
LockIn/                  ← web PWA (this repo)
  src/hooks/useAuth.ts       Google OAuth session
  src/hooks/useNotifications.ts  Browser/SW push notifications
  src/lib/cloudStorage.ts    Supabase read/write helpers
  src/lib/supabase.ts        Supabase client (null-safe if .env missing)
  supabase/schema.sql        Run once in Supabase SQL Editor

lockedin-mobile/         ← Expo Go app (not started yet)
  Same Supabase project → same data, same auth
```

```
Sync flow (web):
  Login → cloudFetchAll() → write all days to localStorage → UI updates
  Edit  → localStorage instantly → cloudUpsertDay() after 800ms debounce
  Tab focus → cloudFetchDay(today) → merge into state
```

```
Notification flow (web):
  Timer hits zero → onSessionComplete() → browser Notification API
                                        → SW showNotification() (works backgrounded)

Notification flow (mobile, planned):
  Timer starts → scheduleNotificationAsync({ seconds: N })
              → APNs fires even if phone is locked/app closed
```
